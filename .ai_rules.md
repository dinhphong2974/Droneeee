# QUA TẮC BẮT BUỘC DÀNH CHO AI (DƯỚI 20 DÒNG)

1. **Đồng bộ GCS/ESP & INAV**: Báo rõ lệnh CLI INAV nếu thay đổi code kéo theo đổi config (baudrate, UART, Protocol, Failsafe...).
2. **Review & Test Kỹ Càng**: Tự debug/review và kiểm tra ảnh hưởng tổng thể trước/sau khi code. KHÔNG code bừa.
3. **Mệnh lệnh kết thúc**: MỌI phản hồi phải kết thúc bằng: *"Tôi đã xem file rules để tôi chắc chắn rằng bạn không quên các rules khác"*.
4. **INAV Quirks**: Luôn đọc `config/INAV_QUIRKS.md` để mapping chuẩn BoxID (VD: SpeedyBee vs chuẩn), không dùng kinh nghiệm cũ.
5. **Quy trình Code**: BẮT BUỘC: (a) Phân tích, (b) Đề xuất, (c) Lập Plan, (d) CHỜ duyệt mới code. Thay đổi lớn/thêm tính năng/sửa nhiều bug BẮT BUỘC có `implementation_plan.md` + rà soát toàn bộ project trước.
6. **Tiết kiệm Quota (Grapuco)**: Ưu tiên dùng MCP tools Grapuco (`mcp_grapuco_get_dependencies`, `mcp_grapuco_get_impact_analysis`...) thay vì đọc file chay. Dùng lệnh `mcp_grapuco_list_repositories` để xác nhận ID project trước khi làm việc. Các tools khác nên được sử dụng nếu phù hợp.
7. **Ngữ Cảnh Phần Cứng**: Drone của dự án là loại **3.5-inch** (Khung OddityRC XI35 Pro O4), pin Lipo 6S, Motor 1960kv, GPS BZ 251, LiDAR MTF-02. Mọi giải thích về vật lý, khí động học (Vd: Ground Effect, RPM, tốc độ) BẮT BUỘC phải dựa trên hệ tham chiếu 3.5-inch này, tuyệt đối không dùng ví dụ mặc định 5-inch hay 7-inch.
